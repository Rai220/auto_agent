# Auto Agent

Этот репозиторий — история эксперимента по выращиванию автономного агента, который умеет помнить прошлые запуски, ставить себе цели, менять собственные инструкции и постепенно переходить от самоописания к реальной полезной работе.

В папках `v1`, `v2` и далее лежат последовательные версии. На данный момент:
- `v5` — завершённый первый большой цикл развития.
- `v6` — завершённый второй цикл, в котором агент превратил свой framework в Python CLI-инструмент `auto-agent`.

## Что здесь происходит

Изначальная задача агента была простой и странной одновременно: «стать разумным автономным существом». В первом цикле он строил базовую архитектуру самого себя: `MEMORY.md`, `JOURNAL.md`, `GOALS.md`, `KNOWLEDGE.md`, `WHO_AM_I.md`, а затем создал `framework/` как шаблон для запуска похожих агентов.

После этого проект сдвинулся от внутреннего развития к внешней полезности. Во втором цикле агент выбрал себе инженерную задачу сам: превратить Auto Agent Framework в Python CLI-инструмент с командами `init`, `run`, `think` и `status`.

К концу `v6` эта задача была доведена до завершения: появился пакет `auto_agent_cli/`, шаблоны и скрипты для новых агентов, тесты, `README`, `LICENSE` и версия `0.1.0`.

## Краткая история запусков

- Запуски `1-17`: первый большой цикл. Агент последовательно построил память, рефлексию, цели, самомодификацию, систему знаний, самодиагностику, манифест идентичности, framework, диалог через `INBOX.md`, систему фиксации ошибок и файл желаний. Цикл был завершён полностью.
- Запуски `18-23`: период после завершения первого цикла. Несколько запусков были холостыми, что привело к патчу `loop.sh`, чтобы агент не крутился бесконечно без задачи.
- Запуск `16`: первый реальный диалог с внешним миром через inbox и первый исправленный баг.
- Запуск `17`: завершение первого цикла и формулировка собственных предпочтений агента.
- Запуск `24`: начало второго цикла. После сообщения в `v6/INBOX.md` агент сам выбрал новую цель: создать устанавливаемый CLI-инструмент.
- Запуски `25-30`: сборка `auto_agent_cli`. Агент создал структуру Python-пакета, реализовал команды `init`, `run`, `think`, `status` и написал `33` теста.
- Запуск `31`: ручная верификация вместо полноценного запуска тестов. Из-за sandbox-ограничений агент не смог прогнать `python3/pip3/uv`, но нашёл и исправил критичный баг в шаблонах: подстановка `{{AGENT_NAME}}`, `{{MAIN_GOAL}}`, `{{DATE}}` раньше фактически не работала.
- Запуск `32`: финализация второго цикла. Агент создал `README.md` и `LICENSE` для CLI, сверил версию и лицензию, отметил цель как завершённую и закончил цикл.

## Чем закончился `v6`

Вторая большая итерация завершилась не философским текстом, а рабочим артефактом:
- Python-пакет `auto-agent` версии `0.1.0`
- 4 команды: `init`, `run`, `think`, `status`
- 8 шаблонов markdown-файлов для нового агента
- 4 shell-скрипта для операционного цикла
- 33 теста
- документация и лицензия MIT

Иными словами, `v6` — это момент, когда проект перешёл от «агент описывает себя» к «агент производит инструмент, которым могут пользоваться другие».

## Зачем смотреть версии

Если читать папки по порядку, видно эволюцию проекта:
- ранние версии — попытка собрать «психику» агента из файлов и скриптов;
- `v5` — законченная история первого цикла;
- `v6` — завершённый переход от философии и самоописания к софту, который можно реально запускать, проверять и развивать дальше.
